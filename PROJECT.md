## What this is

A Solana DePIN VPN. Home nodes carry traffic; users pay in the launch token ($WEFT); a single
Anchor program (`weft`) holds custody, settlement, staking, and node identity (Bubblegum V2
compressed NFTs). Off-chain services meter usage and settle in compact on-chain checkpoints.

- On-chain: one program, instructions for `initialize_core`, escrow deposit/pay/withdraw,
  `stake`/`unstake`, `post_epoch`, `claim`, `dispute`, `register_node` (cNFT). Vaults are PDAs.
- Off-chain (ESM, @solana/kit): **control-plane** (per-wallet VPN link, quota from escrow),
  **aggregator** (epoch poster — posts each epoch's reward root on-chain + serves merkle proofs),
  **indexer** (node directory), **buyback-burn** (SOL→$WEFT→burn worker, off by default).
  **registry-provision** = init + cNFT provisioning + node agent.
- Frontend: `weft-web/cabinet` (Vite, Vercel, base `/app/`). Reads mint/vault/treasury/decimals
  from the on-chain Distributor **at runtime** — no rebuild when the token launches.
- Economics: every $WEFT payment splits **70% nodes / 20% burned on-chain (BurnChecked) / 10% treasury**.
- **Node rewards are claimed ON-CHAIN from the reward vault** (the 70% lands there on every payment).
  Nodes claim their share with a merkle proof (permissionless, solvency-guarded by `post_epoch`,
  double-claim-proof via a per-(epoch,node) `ClaimStatus` PDA). This is the **single** payout rail —
  there is no off-chain hot-wallet payout, so no manual funding and no double-pay surface.

## Security + correctness audit (2026-07-04) — hardening shipped

A full 5-dimension audit (program, economics, off-chain services, Token-2022, deploy/secrets) ran
after the aborted launch. Confirmed findings fixed (all verifiable in source; 46 Rust + 95 TS tests green):

- **Payout architecture (was double-rail):** removed the off-chain `/withdraw-earned` hot-wallet rail
  entirely; node rewards are now the single on-chain claim from the vault. The cabinet `Rewards`
  panel claims on-chain (proof from `/claimable` → `getClaimInstruction`, skipping already-claimed
  epochs via the `ClaimStatus` PDA). This eliminated the double-pay / stranded-funds / non-idempotent
  payout class of bugs at the root.
- **Program:** `deregister_node` now refuses while stake remains (was: closed the PDA and locked the
  stake forever); `dispute` slashing is bounded to ≤25% of the position per call (`MAX_SLASH_BPS`),
  reputation-first (was: whole stake seizable on one signature); claim timestamp uses `checked_add`;
  `InitializeCore.treasury` pinned to `token::mint = reward_mint`.
- **Secrets:** the Reality private key, founder UUID, short-id, and relay token were committed
  (public repo + bundle) — **de-committed**. Fresh values are generated by
  `scripts/gen-relay-secrets.mjs` into a local uncommitted `~/.config/weft/relay-secrets.env`; the
  cutover sources them. The old values are **burned** — rotate the VPS Reality/Xray identity at rebrand.
- **Aggregator epoch pipeline:** `/receipts` now requires `WEFT_RECEIPTS_TOKEN`; the settle epoch is
  derived from on-chain `current_epoch` (no store-poisoning, no gaps); a reentrancy guard + post-first
  cursor ordering make settlement crash-replayable; request bodies are capped.
- **Control-plane:** `/provision` no longer persists zero-escrow wallets (was: unbounded user
  minting); the metering tick writes once per cycle (was O(N²)); settlement is atomic + idempotent +
  boot-swept (a crash can't strand a paid tab); `/relay/live` is cached; boots assert
  `WEFT_MINT == distributor.rewardMint`.
- **Token-2022:** the init extension guard fails **closed** if the RPC can't return parsed extensions
  (was: fee/hook mint could slip through); `scaledUiAmountConfig` denylisted; decimals ≥ 6 enforced;
  the frontend token-program cache never poisons on a transient miss.
- **Buyback:** removed the `.unref()` that killed the worker after one cycle; store writes atomically.
- **Deploy/gating:** `build-web.sh` requires `VITE_WEFT_CLUSTER` and asserts the mainnet bundle
  carries no devnet RPC; the cutover fails loudly if the aggregator doesn't come up (stale-id bundle);
  preflight dev-gate greps fixed (`PRODUCTION_UI`); `scripts/deploy-program.sh` wraps the program
  deploy with a cluster echo + typed confirm.

### Known, documented limitations (not launch blockers)

- **One escrow across multiple relays:** quota is derived from the wallet's global escrow and the
  settlement-signature dedup is per-node-local, so with 2+ independent relay nodes a wallet could get
  N× service for 1× escrow. Not exploitable in the current single-relay topology; scope per-node
  escrow/settlement before running multiple independent relays.
- **Program medium items** (pause gates only `register_node` not money moves; no clawback sweep;
  `shutdown_core` doesn't verify stake wound down) are hardening for a later pass — the program
  redeploys fresh at rebrand, so they can be folded in then.

## The Token-2022 incident — post-mortem (do not repeat)

**What happened:** at the coordinated launch, the token minted on pump.fun was **Token-2022**
(owner `TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb`), not classic SPL. The on-chain program was
already token-agnostic (uniform `token_interface`), but **every off-chain client hardcoded the
classic Token program**, so the launch could not proceed on the real token and the window was lost.

**Root cause:** assuming the token program instead of detecting it. pump.fun now mints Token-2022 by default.

**Fix (shipped):** the whole stack now reads the reward mint's **owner program from chain at
runtime** and threads it into every ATA derivation, transfer, burn, and instruction:
- `registry-provision/src/initCore.ts` — guard accepts classic **or** Token-2022, **rejects**
  economy-breaking extensions (transfer fee, transfer hook, permanent delegate, default-frozen,
  interest, non-transferable, confidential), and passes the detected `tokenProgram` to init.
- `aggregator` (`cli.ts` resolves the owner → `payout.ts`, `pay.ts` PayConfig, `server.ts`).
- `cabinet/src/lib/chain.ts` — memoized `resolveTokenProgram(mint)`; `ata()`, `rewardConfig()`,
  deposit/settle/withdraw and `Staking.tsx` stake all token-program-aware.
- `control-plane/src/faucet.ts` (devnet test mint) and `buyback-burn` (web3.js v1) too.

**Proven:** full money flow rehearsed on a local validator with a real Token-2022 mint —
`core:init → deposit → pay` produced the exact **70% vault / 10% treasury / 20% on-chain burn
(mint supply decrease)**, and `TokenPayout` credited a node (ATA auto-created). CI regression test
in `aggregator/test/pay.test.ts` asserts the settlement builders never fall back to classic.

## Relaunch checklist (at the rebrand — mainnet spend happens here)

The program is fully T2022-safe; a relaunch is a **fresh deploy + id rotation**, not a code change.
1. New program keypair; `declare_id!` (`programs/weft/src/lib.rs:19`) + `Anchor.toml` (3 clusters).
   Build with `anchor build --ignore-keys` (never commit the new secret).
2. Regenerate **both** SDKs from the new IDL: `pnpm codegen` (Weft/sdk) **and** the separate
   `weft-web/sdk` (`codegen.mjs`). Then `pnpm -r build` (rebuilds services + the id-baked bundles).
3. **Rebuild + recommit the operator bundles** — nodes `curl` them from GitHub raw:
   `services/{aggregator,control-plane,registry-provision}/dist/*.mjs`. Grep the tree for the OLD
   id before deploying (past landmine: a stale `HFt8Bm7r…` id lived in a committed bundle).
4. Deploy the program with `./scripts/deploy-program.sh mainnet-beta` (echoes cluster + id + admin
   balance, typed confirm). Generate relay secrets once:
   `node scripts/gen-relay-secrets.mjs > ~/.config/weft/relay-secrets.env && chmod 600 ~/.config/weft/relay-secrets.env`.
   Then `./scripts/mainnet-launch.sh <CA> --yes` (`core:init` detects the CA's token program) →
   `./scripts/mainnet-cutover.sh <CA> --yes` (derives the distributor PDA from the SDK — rotation-safe;
   sources the relay secrets). **No hot-wallet funding** — node rewards are claimed on-chain from the vault.
5. Rebuild the cabinet from mainnet env (`VITE_WEFT_CLUSTER=mainnet-beta` — the build now requires it
   and asserts no devnet RPC leaks); redeploy to Vercel; verify `/price` + `/app`. The reward **mint**
   needs no rebuild (read at runtime), but a rotated **program id** is build-time-baked into the cabinet,
   both SDKs, and every bundle — those MUST be regenerated/rebuilt (steps 1–3) on any id change.

**Rebrand swap-list** (keep "Weft" until the new brand lands): collection name + URI in
`registry-provision/src/provision.ts` (`'Weft Nodes'`, `COLLECTION_URI`), node metadata +
symbol in `registry-provision/src/nft.ts` (`NODE_METADATA_URI`), domain via `WEFT_HOST` /
`VITE_WEFT_RELAY_HOST` env (default `vpn.weftnetwork.net`), and the hosted `nft/*.json`.
